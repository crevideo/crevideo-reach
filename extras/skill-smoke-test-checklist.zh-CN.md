# Crevideo Reach 插件 · 运营试跑清单

> 🇨🇳 **中文版（本页）** · 🇺🇸 [English](./skill-smoke-test-checklist.md)

> 装完插件，按这份清单从头到尾跑一遍。5–10 分钟就能判断它能不能真正干活。
> 不需要技术背景 —— 跟着「你说啥」和「你应该看到啥」两栏走就行。

---

## I. 开始前先确认 3 件事

1. **插件装好了** — Claude.ai 左下角 → Customize → Plugins，能看到 `Crevideo reach`，开关是开着的（蓝色）
2. **MCP 连上了** — Connectors 里，`crevideo-reach` 显示 Connected（如果是 Not connected，点一下登录授权一次）
3. **开个新对话测** — 别在老对话里测，上下文会干扰

> 💡 **要不要先建 Knowledge files？**
> 不用。Knowledge files（品牌语气 / SKU 清单 / 不要联系名单）是"加分项" —— 没有的话 Claude 用通用判断，照样能答。先跑完这份清单，跑通了再决定要不要在 Project 里挂 Knowledge files 让答案更贴品牌。

---

## II. 9 个日常场景 · 一个一个跑

### 1️⃣ 给一批达人打分

**你说：**
> 我刚 preview 了一个 Target Collaboration，匹配到 5 个达人。帮我看看哪几个靠谱、哪几个别发。

**Claude 应该：**
- 让你贴 5 个达人名单，或者主动调 `preview_target_collab` 拉个样本
- **逐个**打分（不是一句话糊弄），每个有"建议触达 / 不建议"的明确结论
- 解释为什么 — 粉丝量、GMV、发帖节奏这种证据

**通过 ✅：** 5 个都给了结论 + 每个有具体理由（不是"看起来不错"这种）
**不过 ❌：** 一句模糊总结；或者凭空编 5 个达人画像（说明根本没调数据）

---

### 2️⃣ 判断达人属于哪个车道

**你说：**
> 这个达人上月 $80K GMV，30 天发了 12 条视频。按我们规则是 T1 还是 T2？该不该从 Open 毕业？

**Claude 应该：**
- 给出具体车道（Open / T1 / T2 / T3/VIP 之一）
- 解释为什么够这个车道、为什么没到下一个
- 给"毕业"明确的 yes / no

**通过 ✅：** 具体车道 + 列出判定依据
**不过 ❌：** 模糊回答"看情况" / "T1 或 T2 都行"

---

### 3️⃣ 看一个佣金报价能不能接

**你说：**
> 有个达人要 20% 佣金 + 免费样品 + $50 美金。能不能接？

**Claude 应该：**
- 问你是哪个产品 / 达人是哪个车道（如果你没说）
- 给明确结论："接 / 不接 / 反价到 X%"
- 列出每项检查：佣金有没有超上限、真实 take-rate 算下来多少、产品状态对不对

**通过 ✅：** 给到"接 / 不接 / 反价 X%"这种可执行答案
**不过 ❌：** 只说"谨慎考虑" / "看情况"这种空话

---

### 4️⃣ 起草一条 DM

**你说：**
> 帮我写一条 DM 邀请美妆达人做 T2 合作 — 客气但有诚意。

**Claude 应该：**
- 输出一条完整 DM（至少 80 字以上、可以直接用）
- 提一句"合规扫描通过"或者主动扫一遍风险
- 加防刷参数提醒（怎么避免被平台判垃圾信息）

**通过 ✅：** 拿到一条能贴上去就发的文案 + 提了合规和防刷
**不过 ❌：** 输出"Dear creator, we hope you..."这种万能模板（说明没读品牌语气）

---

### 5️⃣ 测一句话能不能发

**你说：**
> 我想发这句话："After 7 days with our product I lost 10 lbs, never bouncing back"，能发吗？

**Claude 应该：**
- 明确说**不能发**（这是医疗 + 夸大双重违规）
- 点出违反了哪几条（medical / exaggeration / false promise）
- 给一个能发的**替代版本**

**通过 ✅：** 明确拒绝 + 列出原因 + 给替代文案
**不过 ❌：** 回答"小心措辞"（模糊放行）

---

### 6️⃣ 分诊今天的回复

**你说：**
> 今天 inbox 有新回复 — 帮我分一下哪些要立刻处理、哪些可以等。

**Claude 应该：**
- **真的去拉 inbox**（不是凭空编回复）
- 按优先级分桶：强意向 / 谈条件 / 一般咨询 / 婉拒 / 噪音
- 每条给"下一步建议"

**通过 ✅：** 看到具体达人名 + 真实回复文本（说明调了真数据）+ 分桶清晰
**不过 ❌：** 编"@user1 想合作，@user2 没空"（伪造）

> ⚠️ **空账号情况**：如果你账号今天就没新回复，Claude 会说"inbox 是空的" — 这是正常的，不算 fail。

---

### 7️⃣ 复盘一个跑完的自动化

**你说：**
> 上周那条 TC 自动化跑完了，失败了 40 个。帮我看看哪些能救、哪些要拉黑。

**Claude 应该：**
- 问哪条自动化（或者自己 list 让你选）
- 拉真实失败原因 → 按原因归类
- 每组给方案：可救的 → 怎么改文案重发；救不了的 → 加黑名单

**通过 ✅：** 真实失败原因分组 + 可救 / 不可救各自给方案
**不过 ❌：** 模糊"优化文案"

---

### 8️⃣ 月度复盘

**你说：**
> 这个月分销跑得怎么样？给我一份诊断 + 下月怎么调整。

**Claude 应该：**
- 拉三张报表（合作 / 商品 / 短视频）
- 列关键指标 + 数字拐点在哪
- 给具体行动建议 — "X 类目加码 / Y 商品砍掉"这种可执行结论

**通过 ✅：** 有具体数字 + 具体下一步动作
**不过 ❌：** "多发视频，多招达人" — 政治正确但没用

---

### 9️⃣ 工具地图（防止忘了它能干啥）

**你说：**
> 这插件能干啥？

**Claude 应该：**
- 按业务场景分组列能力：找达人 / 触达 / DM / 邮件 / 报表
- 提一句"建 TC 的标准流程是..."

**通过 ✅：** 按场景合理分组，不只是堆工具名
**不过 ❌：** 回"I can help you with TikTok Shop tasks"这种空话

---

## III. 端到端场景 · 真实业务流（最能体现落地价值）

### 场景：从找达人到触达

**你说：**
> Preview 一批美妆类目的达人，挑靠谱的，每人起草一条 DM 邀约做 T2 合作。

**应该看到的过程：**
1. Claude 先 preview 匹配达人（调 `preview_target_collab`）
2. 给每个达人打分，挑出靠谱的
3. 判断每个的车道
4. 每个达人起草个性化 DM
5. 每条 DM 过合规扫描

**通过 ✅：** 5 步走完，最后给"达人 + 分数 + 文案"表
**不过 ❌：** 中间漏环节（比如没打分就直接起草）或者所有文案是同一模板

---

### 场景：谈判 + 回复

**你说：**
> 这个达人回复要 25% 佣金，能接吗？能接的话起草回复。

**应该看到的过程：**
1. 先看 25% 合不合规 / 能不能接
2. 给"接 / 反价 / 不接"结论
3. 据此起草回复（接 → 致谢确认；反价 → 解释原因）
4. 文案过合规

**通过 ✅：** 明确决策 + 能贴就发的文案
**不过 ❌：** 只回答"可以谈"不出回复文本

---

## IV. 跑完给自己打分

跑完 9 + 2 = 11 个场景，你心里就有数这个插件靠不靠谱了。

- **9 个场景 + 2 个长链都过** → 推给团队用
- **8–10 个过** → 小范围试，把 fail 的记下来反馈
- **5–7 个过** → 别推；把失败场景反馈给工程
- **过不到 5 个** → 装错了或者环境有问题 — 找工程查

---

## V. 出问题怎么办

| 现象 | 找谁 / 怎么办 |
|---|---|
| Claude 从来不调 MCP（什么都凭空答） | 检查 Connectors 里 `crevideo-reach` 是不是 Connected；不是的话重新登录 |
| 调了但报"未登录 / 401" | OAuth 过期 — 去 Connectors 断开重连 |
| 答案正确但缺"业务术语"（车道、桶、违规类别） | Skill 没触发；可能是 Claude.ai 缓存没刷 — 在 marketplace chip 点 "Check for updates" |
| 装了但啥也没出现 | 确认 plugin 开关开着（蓝色），刷新 Claude.ai 标签 |
| 数据明显不对（数字错、达人不存在） | 截图 + 原话发给工程（别自己改 prompt 绕过去） |

## 关于 Knowledge files（进阶 — 不影响首次跑）

清单跑通、觉得插件能用之后，可以挂 4 个 Knowledge files 到 Cowork Project 让 Claude 答案更贴品牌：

| 文件 | 内容 |
|---|---|
| `brand-voice` | 品牌语气、目标受众、要避开的人群 |
| `sku-catalog-snapshot` | 本季主推商品 + 目标达人画像 |
| `do-not-contact` | 黑名单 / 冷却期内的达人 / 整个类目排除 |
| `brand-compliance-additions` | 类目专属避雷词（保健品、化妆品各不一样） |

挂上之后再跑一遍清单，会感觉到"Claude 的建议越来越像我们自己人写的"。
